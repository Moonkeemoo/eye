## 1. Середовище: WSL2, Docker, Ubuntu

### WSL2 — це не Docker і не друга ОС
**WSL2** (Windows Subsystem for Linux, версія 2) — вбудована в Windows технологія від Microsoft: легка Linux-машина, що працює **пліч-о-пліч** з Windows, ділить із ним файли й мережу. Це **не** дуалбут (жодного ребуту в іншу ОС) і **не** важка віртуалка типу VirtualBox.

### Звідки взявся `docker-desktop`
Docker Desktop при встановленні створює *для власних потреб* службовий WSL2-дистрибутив `docker-desktop`. Це його внутрішня кухня, не середовище для роботи. Те, що він єдиний у списку `wsl -l -v`, означало лише: повноцінного Linux для розробки ще нема.

### Що ми ставимо й що при цьому НЕ відбувається
Команда `wsl --install -d Ubuntu` ставить **окремий** дистрибутив Ubuntu поруч із `docker-desktop`. Вони не перетинаються.

- ❌ Windows не переставляється, нічого не форматується
- ❌ Драйвер NVIDIA лишається у Windows (CUDA сама «пробивається» в Ubuntu — фішка WSL2)
- ❌ Docker не чіпається
- ✅ Ubuntu — ізольована пісочниця. Видаляється однією командою: `wsl --unregister Ubuntu`. Нуль ризику для системи.

### Чому саме Ubuntu / навіщо Linux
Увесь світ ML — **Linux-first**. PyTorch, CUDA, Ultralytics і дослідницький код з GitHub (SAM, DETR) пишуться й тестуються на Linux; команди з туторіалів і статей там просто працюють, а на Windows часто треба «допилювати». Ubuntu — найпопулярніший і найкраще підтримуваний дистрибутив, тому під неї все заточено.

> Альтернатива: YOLOv8 *можна* тренувати й на native Windows (у PyTorch/Ultralytics є Windows-збірки під CUDA). На один шар менше, але далі в CV-подорожі починаються Linux-only тертя. Ми обрали WSL2+Ubuntu як середовище, в якому реально працює CV-інженер.

## 2. PyTorch і CUDA

Тримай у голові «вежу» — знизу залізо, зверху твій код. Сигнал від `python train.py` проходить її згори вниз:

```
┌──────────────────────────────────────┐
│  Ultralytics YOLOv8   ← твій train.py │  високий рівень, 5 рядків коду
├──────────────────────────────────────┤
│  PyTorch              ← рушій навчання│  тензори, autograd, тренувальний цикл
├──────────────────────────────────────┤
│  cuDNN + CUDA toolkit ← «перекладач»  │  бібліотеки під нейромережі (cu121)
├──────────────────────────────────────┤
│  NVIDIA-драйвер (у Windows)           │  доступ до GPU, пробиває CUDA у WSL2
├──────────────────────────────────────┤
│  RTX 4070 Ti SUPER  (16 GB VRAM)            │  залізо, де все рахується
└──────────────────────────────────────┘
```

### CUDA — мова, якою говорять із GPU
**Навіщо GPU.** CPU — кілька дуже розумних робітників, працюють послідовно. GPU — тисячі простих робітників, що роблять одну дію **одночасно**. Навчання нейромережі = мільйони незалежних множень матриць → ідеальна робота для GPU. Тому те, що CPU рахує годинами, GPU — за хвилини.

**Що таке CUDA.** Платформа й інструменти NVIDIA, що дозволяють запускати *будь-які* обчислення (не лише графіку) на їхніх GPU. CUDA — це міст, через який програма каже відеокарті «помнож мені оці матриці». Без CUDA GPU вміє лише малювати на екран.

**cuDNN** — бібліотека NVIDIA поверх CUDA, заточена під нейромережі (згортки, нормалізації). PyTorch кличе її під капотом.

**Версії (`cu121`).** У `pip install torch --index-url .../cu121` маркер `cu121` = CUDA 12.1: «дай збірку PyTorch, що несе бібліотеки під CUDA 12.1». Свіжий драйвер у Windows цю версію підтримує — вежа стикується.

### PyTorch — рушій, який реально вчить мережу
Фреймворк глибокого навчання (Meta). Визначає мережу, проганяє дані, оновлює ваги. Ultralytics/YOLO — обгортка поверх нього; `model.train()` усередині = PyTorch.

**1. Тензор.** Базова одиниця даних — багатовимірний масив чисел. Фото кружки → тензор `[3, 640, 640]` (3 канали кольору × 640 × 640 пікселів). Ключове: тензор уміє жити **на GPU**. `.to('cuda')` (або `device=0`) перекидає його з оперативки на відеокарту — це і є міст до CUDA.

**2. Autograd (автоматичне диференціювання).** Серце навчання. Мережа дивиться на фото → дає прогноз → рахує помилку (loss) → треба зрозуміти, *в який бік підкрутити кожну вагу*. Це похідні (градієнти). PyTorch сам запамʼятовує всі операції й рахує градієнти автоматично — жодного матану руками.

**3. Тренувальний цикл.** Повторення чотирьох кроків:
```
forward  → прогнати фото через мережу, отримати прогноз
loss     → виміряти помилку (наскільки рамки/класи неправильні)
backward → autograd рахує градієнти
step     → оптимізатор підкручує ваги в бік меншої помилки
```
Ultralytics ховає цей цикл усередині `model.train()` — але знати, що там саме це, важливо.

### Як це працює в нашому проєкті
```
фото кружок (.jpg)
   → PyTorch робить з них тензори
   → .to(GPU) перекидає на 4070 Ti через CUDA
   → forward/backward у циклі підкручують ваги
   → найкращі ваги зберігаються як best.pt
```
- `device=0` — рахуй на CUDA-GPU №0 = 4070 Ti.
- `torch.cuda.is_available()` — PyTorch бачить CUDA? = вся вежа зібрана. Перша перевірка після setup'у.
- `batch=16`, `imgsz=640` — впираються у **VRAM** (16 GB памʼяті GPU): модель + пачка фото мають влізти. Не влізло → `CUDA out of memory` → знизити `batch`.

## 3. Залізо: розподіл

```
ПК (Windows + WSL2, RTX 4070 Ti SUPER)        MacBook + вебка
   ТРЕНУВАННЯ (важке, на GPU)    ──best.pt──►   ІНФЕРЕНС («морда»), real-time
```
Тренування — там, де потужний GPU. Інференс легкої моделі з однієї вебки потужного GPU не потребує — тому «морда» зручно живе на маку, де ти сидиш. Між ними переноситься один файл `best.pt`. Хмара не потрібна: 16 GB VRAM вистачає для YOLOv8 n/s/m.

## Шпаргалка для співбесіди

> Короткі формулювання «як сказати вголос». Поповнюється.

- **Чому GPU для глибокого навчання?** Бо це паралельні матричні операції; GPU має тисячі ядер і робить їх одночасно, CPU — кілька ядер послідовно.
- **Що таке CUDA?** Платформа NVIDIA для обчислень загального призначення на GPU. PyTorch використовує її через cuDNN.
- **Що таке тензор?** Багатовимірний масив, який може лежати на GPU й підтримує autograd.
- **Як вчиться мережа?** Цикл forward → loss → backward → optimizer step. Backward рахує градієнти автоматичним диференціюванням, оптимізатор оновлює ваги.
- **Що робить `device=0`?** Кладе обчислення на перший CUDA-GPU.
- **Навіщо WSL2/Ubuntu?** ML-екосистема Linux-first; те саме середовище, що в продакшені й у дослідницькому коді.

## Глосарій

| Термін | Коротко |
|---|---|
| **WSL2** | Легка Linux-машина всередині Windows (вбудована, від Microsoft). |
| **Дистрибутив (distro)** | Конкретна збірка Linux (Ubuntu, Debian…). У WSL їх може бути кілька поруч. |
| **CUDA** | Платформа NVIDIA для обчислень на GPU. |
| **cuDNN** | Бібліотека NVIDIA під нейромережі поверх CUDA. |
| **VRAM** | Власна памʼять відеокарти (у нас 16 GB). Обмежує batch/розмір моделі. |
| **Тензор** | Багатовимірний масив; може жити на GPU; підтримує autograd. |
| **Autograd** | Автоматичне обчислення градієнтів у PyTorch. |
| **Градієнт** | Похідна loss по вазі — у який бік і наскільки підкрутити вагу. |
| **Loss** | Міра помилки прогнозу; її мінімізують під час навчання. |
| **Epoch** | Один повний прохід моделі по всьому датасету. |
| **batch** | Скільки фото обробляється за раз. |
| **Transfer learning** | Доналаштування передтренованої моделі під свою задачу (замість навчання з нуля). |
| **`best.pt`** | Файл з найкращими вагами навченої моделі. |
| **Інференс** | Застосування навченої моделі до нових даних (тут — детекція з вебки). |
