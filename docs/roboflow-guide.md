# Roboflow: розмітка, split, аугментації (Етапи 2-3)

Покрокова інструкція від фото до готового датасету для тренування.
Класи: **mug**, **thermos**, **glass** (латиницею, без пробілів — завжди однаково).

---

## 0. Акаунт (один раз)
1. [roboflow.com](https://roboflow.com) → Sign up (безкоштовного тарифу вистачає).
2. Create Workspace (будь-яка назва).

## 1. Проєкт
1. **Create New Project**.
2. Project Type: **Object Detection**.
3. Назва проєкту: `eye` (або будь-яка). Annotation Group: `objects`.

## 2. Залити фото
1. **Upload** → перетягни всі ~300 фото (телефонні JPEG/HEIC підходять).
2. Дочекайся обробки → **Save and Continue**.

## 3. Розмітка (bounding box)
Для кожного фото:
1. Інструмент прямокутника → обведи **кожен** предмет із наших класів.
2. Признач клас (`mug` / `thermos` / `glass`).

Правила якості (це прямо впливає на метрики):
- рамка **щільно** по предмету, без зайвого «повітря»;
- **перекриті** предмети теж розмічай — по видимій частині;
- не пропускай предмети на задньому плані, якщо вони з наших класів;
- **консистентність**: той самий предмет = завжди той самий клас. Неоднозначна розмітка = шум, який модель не подолає.

> 💡 Концепція **IoU** (Intersection over Union) — площа перетину / площа обʼєднання двох рамок (твоєї та передбаченої). На ній стоїть уся оцінка детекції. Деталі — у `handbook.md`.

## 4. Split (train / val / test)
При генерації версії Roboflow ділить дані. Постав **70 / 20 / 10**:
- **train** — на чому модель вчиться;
- **val** — перевірка під час тренування (підбір, рання зупинка);
- **test** — чіпається ОДИН раз у кінці, для чесної фінальної оцінки.

> ⚠️ Окремий test потрібен, щоб не обдурити себе: підбирати модель і фінально оцінювати на тих самих даних = data leakage. Класичне питання на співбесіді.

## 5. Аугментації (штучно роздувають датасет, борються з overfitting)
Обери розумні:
- поворот **±15°**;
- віддзеркалення **по горизонталі**;
- яскравість/контраст **±15-25%**;
- легкий blur / шум.

❌ **НЕ** додавай віддзеркалення по вертикалі — у наших предметів є «верх/низ» (перевернута склянка — інша ситуація, модель заплутається).

## 6. Експорт
1. **Generate** версію датасету.
2. **Export Dataset** → формат **YOLOv8** → **show download code**.
3. Скопіюй сніпет виду:
   ```python
   rf = Roboflow(api_key="xxxxxxxx")
   project = rf.workspace("...").project("...")
   dataset = project.version(1).download("yolov8")
   ```
4. **Передай цей сніпет мені** — я вставлю його в `train.py`, і запускаємо тренування.

## Чек-ліст перед тренуванням
- [ ] ~100 фото на клас, з різноманіттям (ракурси/світло/фон/комбінації)
- [ ] усі предмети розмічені, рамки щільні, класи консистентні
- [ ] split 70/20/10
- [ ] аугментації обрані (без вертикального фліпа)
- [ ] експорт YOLOv8, сніпет скопійований
