# labelImg: локальна розмітка (альтернатива Roboflow)

labelImg — простий десктоп-інструмент: малюєш рамку, він зберігає мітку у YOLO-форматі
поряд із фото. Без акаунта, офлайн. Результат той самий, що й у Roboflow → годує `train.py`.

## Що вже готово
- labelImg встановлено в окреме оточення `~/label` (перевірено).
- WSLg активний (`DISPLAY=:0`) — вікно labelImg покажеться прямо у Windows.
- Папка для фото створена: `data_raw/images/` з готовим `classes.txt` (ceramic_mug / thermos / travel_mug).

## 0. Куди класти фото
Перекинь усі фото з телефону в:
```
eye\data_raw\images\
```
(ця ж папка у WSL: `/mnt/c/Users/tomoo/Documents/GitHub/eye/data_raw/images`)

## 1. Запуск
```bash
wsl -d Ubuntu-24.04
source ~/label/bin/activate
cd /mnt/c/Users/tomoo/Documents/GitHub/eye/data_raw/images
labelImg . classes.txt
```
Відкриється вікно labelImg (через WSLg), з трьома готовими класами у списку.

## 2. Налаштування (один раз, у вікні)
- На лівій панелі є кнопка формату збереження — клацай, доки не покаже **YOLO** (не PascalVOC!).
- Меню **View → Auto Save mode** — увімкни, щоб не зберігати руками щоразу.

## 3. Розмітка
- `W` — створити рамку, обведи предмет.
- Вибери клас (ceramic_mug / thermos / travel_mug).
- `D` — наступне фото, `A` — попереднє.
- Поряд із кожним фото зʼявляється `імʼя.txt` із міткою.

Правила якості (як завжди): рамки щільні; перекриті предмети — по видимій частині; класи консистентні.

## 4. ПОДИВИСЬ на формат — це і є сенс локального шляху
Відкрий будь-який збережений `.txt` (хоч у Блокноті). Побачиш рядки на кшталт:
```
0 0.512 0.439 0.220 0.305
```
Розшифровка: `class_id  x_центр  y_центр  ширина  висота`, де все **нормоване 0..1** (частка від
розміру фото). `class_id` — номер рядка з `classes.txt` (ceramic_mug=0, thermos=1, travel_mug=2).
Ось і весь «секрет» міток детекції — більше там нічого немає.

> 🎤 Для співбесіди: *«мітка YOLO — це текстовий рядок на обʼєкт: клас + нормовані центр і розмір рамки. Тому датасет детекції — це просто пари фото + .txt»*.

## 5. Split + data.yaml
Коли розмітив (хоч 10 фото для проби, хоч усі), збери датасет для YOLO одним скриптом:
```bash
source ~/cv/bin/activate
cd /mnt/c/Users/tomoo/Documents/GitHub/eye
python split_dataset.py --src data_raw/images --out datasets/eye-local
```
Скрипт поділить на train/val/test (70/20/10), створить `data.yaml` і надрукує готовий рядок
`DATA_YAML = "..."` для `train.py`. Далі — тренування.

---
**Аугментації робити НЕ треба** — YOLO додасть їх на льоту під час тренування.
Тобто від labelImg потрібні лише дві речі: рамки і класи. Решту бере на себе `split_dataset.py` і сам YOLO.
